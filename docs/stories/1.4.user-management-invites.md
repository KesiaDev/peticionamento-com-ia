## Story
**As a** admin da organização,
**I want** convidar e gerenciar usuários,
**so that** minha equipe acesse o sistema com as permissões corretas.

## Acceptance Criteria
1. Página `/settings/users` listando membros da organização (nome, email, role, status)
2. Botão "Convidar" que envia email com magic link via Supabase Auth
3. Formulário de convite com campos: email, role (select)
4. Admin pode alterar role de qualquer membro (exceto ele próprio)
5. Admin pode remover membro da organização
6. Permissões: apenas role `admin` acessa esta página
7. RLS policy garante que listagem retorna apenas membros da mesma organização

### Supabase Auth — Padrão de Convite
Existem duas abordagens para envio de convite:

1. **Server-side (recomendado para produção):**
   ```typescript
   // Requer service_role key — usar via Edge Function
   const { data, error } = await supabase.auth.admin.inviteUserByEmail({
     email: 'user@example.com',
     data: { organization_id: orgId, role: 'lawyer' }
   });
   ```

2. **Client-side (alternativa para MVP):**
   ```typescript
   // Usa anon key — funciona client-side
   const { data, error } = await supabase.auth.signInWithOtp({
     email: 'user@example.com',
     options: {
       data: { organization_id: orgId, role: 'lawyer' },
       emailRedirectTo: `${window.location.origin}/accept-invite`
     }
   });
   ```

Para MVP, usar abordagem client-side com `signInWithOtp`. Migrar para Edge Function em iteração futura.
