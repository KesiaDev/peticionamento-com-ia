## Story
**As a** desenvolvedor,
**I want** uma camada de abstração para provedores de LLM,
**so that** eu possa alternar entre OpenAI, Gemini e Claude sem alterar código de negócio.

## Acceptance Criteria
1. Interface TypeScript `LLMProvider` com métodos: `generateDocument()`, `chat()`, `getModels()`
2. Adaptadores implementados: `OpenAIProvider`, `GeminiProvider`, `ClaudeProvider`
3. Factory `createProvider(config)` que retorna o provider correto baseado na config da organização
4. Fallback automático: se provider principal falhar, tenta próximo na lista
5. Edge Function `ai-generate` que recebe prompt + provider config e retorna resposta
6. API keys armazenadas como Supabase Secrets (nunca no frontend)
7. Types exportados em `src/types/ai.ts`
8. Migration `00004_create_documents.sql` cria tabela `documents` (id, organization_id, case_id, client_id, type, title, content, llm_provider, llm_model, prompt_used, tokens_used, status, storage_path, created_at, created_by) com RLS
9. Migration `00005_create_ai_usage_log.sql` cria tabela `ai_usage_log` (id, organization_id, profile_id, provider, model, tokens_input, tokens_output, cost_estimated, document_id, prompt_summary, created_at) com RLS

### Supabase Edge Functions — Estrutura Deno

Edge Functions rodam em **Deno runtime**, não Node.js. Imports usam URLs ou `npm:` prefix:

```typescript
// supabase/functions/ai-generate/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

// Para SDKs de LLM no Deno:
import OpenAI from "https://esm.sh/openai@4";
import Anthropic from "https://esm.sh/@anthropic-ai/sdk@0.26";
import { GoogleGenerativeAI } from "https://esm.sh/@google/generative-ai@0.12";

serve(async (req: Request) => {
  // Verificar CORS
  if (req.method === 'OPTIONS') {
    return new Response('ok', { headers: corsHeaders });
  }

  // Verificar auth
  const authHeader = req.headers.get('Authorization');
  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_ANON_KEY')!,
    { global: { headers: { Authorization: authHeader! } } }
  );
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return new Response('Unauthorized', { status: 401 });

  // API keys vêm de Supabase Secrets (Deno.env)
  const openaiKey = Deno.env.get('OPENAI_API_KEY');
  // ...
});
```

### LLM Provider Interface — Detalhes

```typescript
// src/types/ai.ts
export interface LLMProvider {
  id: LLMProviderId;
  name: string;
  generateDocument(params: DocumentGenerationParams): Promise<GeneratedDocument>;
  chat(messages: ChatMessage[]): Promise<ChatResponse>;
  getModels(): Promise<LLMModel[]>;
}

export interface LLMModel {
  id: string;
  name: string;
  maxTokens: number;
  costPer1kInput: number;   // USD
  costPer1kOutput: number;  // USD
}
```

### Fallback — Lógica

O fallback tenta providers na ordem configurada. Exemplo de config da organização:

```typescript
const config: LLMProviderConfig = {
  provider: 'openai',
  model: 'gpt-4o',
  fallbackProviders: ['gemini', 'claude']
};
```

Se OpenAI falhar (timeout, 429, 500), tenta Gemini. Se Gemini falhar, tenta Claude. Se todos falharem, lança `AllProvidersFailedError`.

### RLS Pattern para migrations

```sql
-- Padrão RLS para todas as tabelas:
ALTER TABLE {table} ENABLE ROW LEVEL SECURITY;

-- Policy SELECT
CREATE POLICY "{table}_select_org" ON {table}
  FOR SELECT USING (
    organization_id = (SELECT organization_id FROM profiles WHERE id = auth.uid())
  );

-- Policy INSERT
CREATE POLICY "{table}_insert_org" ON {table}
  FOR INSERT WITH CHECK (
    organization_id = (SELECT organization_id FROM profiles WHERE id = auth.uid())
  );

-- Policy UPDATE
CREATE POLICY "{table}_update_org" ON {table}
  FOR UPDATE USING (
    organization_id = (SELECT organization_id FROM profiles WHERE id = auth.uid())
  );

-- Policy DELETE
CREATE POLICY "{table}_delete_org" ON {table}
  FOR DELETE USING (
    organization_id = (SELECT organization_id FROM profiles WHERE id = auth.uid())
  );
```
