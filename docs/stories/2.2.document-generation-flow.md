## Story
**As a** advogado,
**I want** gerar um documento juridico informando tipo, dados do caso e fatos,
**so that** eu obtenha uma peca juridica redigida pela IA.

## Acceptance Criteria
1. Pagina `/ai/new` com wizard de 3 etapas: Tipo -> Dados -> Resultado
2. Step 1: Selecao do tipo de documento (peticao inicial, recurso, contrato, etc.)
3. Step 2: Formulario contextual com campos: partes envolvidas, fatos, fundamentacao desejada, tribunal, vara
4. Step 3: Loading state -> documento gerado exibido com formatacao
5. Botao "Salvar rascunho" persiste no banco (tabela `documents`)
6. Botao "Editar" abre o editor (Story 2.3)
7. Registro de uso em `ai_usage_log` (provider, model, tokens, custo estimado)
8. Validacao Zod em todos os inputs do formulario

### Chamada ao Edge Function

```typescript
// src/hooks/useDocumentGeneration.ts
import { supabase } from '@/lib/supabase';

export function useDocumentGeneration() {
  return useMutation({
    mutationFn: async (params: DocumentGenerationParams) => {
      const { data, error } = await supabase.functions.invoke('ai-generate', {
        body: {
          documentType: params.type,
          context: params.context,
          providerConfig: params.providerConfig
        }
      });
      if (error) throw error;
      return data as GeneratedDocument;
    }
  });
}
```

### Tabela de custos estimados por modelo

```typescript
// src/lib/ai/pricing.ts
export const LLM_PRICING: Record<string, { input: number; output: number }> = {
  'gpt-4o':            { input: 0.005,  output: 0.015 },
  'gpt-4o-mini':       { input: 0.00015, output: 0.0006 },
  'gemini-1.5-pro':    { input: 0.00125, output: 0.005 },
  'gemini-1.5-flash':  { input: 0.000075, output: 0.0003 },
  'claude-sonnet-4-20250514': { input: 0.003, output: 0.015 },
  'claude-3-5-haiku':  { input: 0.001,  output: 0.005 },
};

export function estimateCost(model: string, inputTokens: number, outputTokens: number): number {
  const pricing = LLM_PRICING[model];
  if (!pricing) return 0;
  return (inputTokens / 1000) * pricing.input + (outputTokens / 1000) * pricing.output;
}
```

#### 4. UI Quality

- Dark theme: uses `text-foreground`, `bg-primary/10`, `text-muted-foreground`, `prose-invert`

#### 5. Pattern Consistency

- Component structure: page -> wizard container -> step components

#### 6. Concerns (LOW severity, non-blocking)

1. **`handleFormSubmit` useCallback deps** — `selectedJurisprudence` is used inside the callback but not in the dependency array. This means the callback captures a stale reference. **Severity: LOW** — in practice the user submits the form once, but it could cause bugs if the user goes back, changes jurisprudence, and re-submits.

2. **Content rendering** — The generated document content is rendered via `whitespace-pre-wrap` plain text, not Markdown/HTML. The AC says "documento gerado exibido com formatacao" — the current approach works but doesn't parse headings/bold/lists from the AI output. **Severity: LOW** — functional but could look plain.

3. **Edge Function prompt construction** — The `prompt` field sent to the Edge Function is built in `useDocumentGeneration` as just the body fields, but the Edge Function also receives `documentType` and `context` separately without using them for prompt construction — it relies on the `prompt` string. Slight redundancy but not harmful. **Severity: INFORMATIONAL**
