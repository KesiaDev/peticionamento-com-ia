## Dev Notes

### Prompt Engineering para Templates Juridicos
- Cada template e um prompt de sistema (system prompt) que sera enviado ao provider de IA junto com os dados do caso
- A estrutura do prompt deve seguir o padrao: contexto geral → tipo de peca → secoes obrigatorias → instrucoes de formatacao → placeholders para dados dinamicos
- Linguagem juridica padrao brasileira (ex: "Excelentissimo Senhor Doutor Juiz de Direito", "Egrégio Tribunal")
- Templates devem ser agnósticos ao provider (OpenAI, Gemini, Claude) — texto puro com placeholders

### Arquivos e Paths
- Diretorio de templates: `src/lib/ai/templates/`
- Tipos compartilhados: `src/lib/ai/templates/types.ts`
- Registry/index: `src/lib/ai/templates/index.ts`
- Componente seletor: `src/components/documents/TemplateSelector.tsx`

### Padroes do Projeto
- TypeScript strict + validacao Zod para inputs dos templates
- shadcn/ui para componentes do seletor
- Tailwind para estilos
- Imports absolutos conforme configuracao do projeto

### Testing
- Framework: Vitest + Testing Library
- Testes de templates: `src/lib/ai/templates/__tests__/`
- Testes de componente: `src/components/documents/__tests__/TemplateSelector.test.tsx`
- Validar que cada template possui todas as secoes obrigatorias
- Validar que placeholders `{{jurisprudencia}}` e `{{legislacao}}` estao presentes
- Snapshot tests para garantir estabilidade da estrutura dos prompts
