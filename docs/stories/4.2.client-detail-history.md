### Supabase Storage — Configuração do Bucket
```sql
-- No migration ou via dashboard
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES (
  'client-documents',

```

**Política RLS para Storage:**
```sql
-- SELECT (download)
CREATE POLICY "Users can view org files"
ON storage.objects FOR SELECT
USING (
  bucket_id = 'client-documents'
  AND (storage.foldername(name))[1] = (auth.jwt()->>'organization_id')
);

-- INSERT (upload)
CREATE POLICY "Users can upload org files"
ON storage.objects FOR INSERT
WITH CHECK (
  bucket_id = 'client-documents'
  AND (storage.foldername(name))[1] = (auth.jwt()->>'organization_id')
);

-- DELETE
CREATE POLICY "Users can delete org files"
ON storage.objects FOR DELETE
USING (
  bucket_id = 'client-documents'
  AND (storage.foldername(name))[1] = (auth.jwt()->>'organization_id')
);
```

### Upload Pattern
```typescript
async function uploadFile(clientId: string, file: File, description?: string) {
  const orgId = getCurrentOrganizationId();
  const storagePath = `${orgId}/${clientId}/${Date.now()}_${file.name}`;

  // 1. Upload to Storage
  const { error: uploadError } = await supabase.storage
    .from('client-documents')
    .upload(storagePath, file, { contentType: file.type });

  if (uploadError) throw uploadError;

  // 2. Insert metadata in client_files
  const { error: dbError } = await supabase
    .from('client_files')
    .insert({
      client_id: clientId,
      organization_id: orgId,
      file_name: file.name,
      file_type: file.type,
      file_size: file.size,
      storage_path: storagePath,
      description,
    });

  if (dbError) throw dbError;
}
```
